Introduction to Statistics
CO: 1 · Data Analysis & Descriptive Methods
1.1 Introduction & Data Analysis REMEMBER
IPL APPLICATION · DATASET STRUCTURE

The IPL predictor ingests two primary datasets — matches.csv (one row per match, 2008–2025) and deliveries.csv (one row per ball bowled). Understanding how raw data is collected, structured and classified is the first step of any statistical analysis.

Types of data in our dataset: Match results are categorical (win/loss); runs scored are discrete quantitative; bowling economy rate is continuous quantitative; toss decision is nominal; match year is ordinal.

Pipeline: Raw CSVs → cleaning (removing no-result / DLS matches) → feature engineering (run rate, win%) → statistical modelling → prediction output.

1.2 Measures of Central Tendency UNDERSTAND
IPL APPLICATION · TEAM SCORING AVERAGES

The Poisson model's core parameter λ is the mean run-rate across all historical innings. Central tendency measures tell us where a team's scoring is "centred."

// Mean run-rate (used as λ in Poisson model): x̄ = (1/n) × Σ (runs_i / 20) // over all innings i // Median: robust to blowout innings (e.g. 250+ totals) M = middle value of sorted run-rates // Mode: most frequent score bracket // Used to identify a team's "typical" total

Why mean over median? Poisson uses arithmetic mean as its MLE estimator. However for skewed innings distributions the median gives a more robust picture. We check both in the data quality step.

1.3 Measures of Dispersion UNDERSTAND
IPL APPLICATION · SCORE VARIABILITY & CONSISTENCY

Dispersion measures quantify how consistent a team's scoring is. A team with low variance is predictable; high variance teams are "boom or bust." Both affect prediction confidence intervals.

// Sample variance of run-rates: s² = Σ(xᵢ - x̄)² / (n-1) // Standard deviation: s = √s² // Coefficient of variation (dimensionless consistency): CV = (s / x̄) × 100% // lower = more consistent // Poisson model: SD of predicted score = √(λ × 20) // Because Poisson variance = mean = λ
1.4 Moments · Skewness · Kurtosis UNDERSTAND
IPL APPLICATION · SHAPE OF SCORE DISTRIBUTIONS

IPL scores are rarely symmetric. Teams batting at Wankhede (batting-friendly) show left-skewed distributions; spin-friendly Chennai shows right-skewed distributions (scores cluster low).

// r-th central moment: μᵣ = (1/n) Σ (xᵢ - x̄)ʳ // Skewness (3rd standardised moment): γ₁ = μ₃ / σ³ // +ve → right-skewed (low scores), -ve → left-skewed // Kurtosis (4th standardised moment): γ₂ = μ₄ / σ⁴ - 3 // High kurtosis → extreme scores occur more than Normal predicts // Important for adjusting venue factor in the model

Random Variables
CO: 2 · Probability Theory & Stochastic Modelling
2.1 Introduction to Random Variables UNDERSTAND
IPL APPLICATION · MATCH OUTCOMES AS RANDOM VARIABLES

A random variable (RV) maps outcomes of a random experiment to numbers. Every quantity computed by the predictor is a realisation of an RV.

Discrete RVs in our model: X = runs off a single delivery {0,1,2,3,4,6}; Y = wickets in an innings {0…10}; W = match winner {0,1}.

Continuous RVs: λ = team run-rate (ℝ⁺); θ = win probability ∈ [0,1] modelled by the Beta distribution; venue win rate ∈ [0,1].

The distinction matters: discrete RVs use PMFs; continuous RVs use PDFs — both are core to the models on this page.

2.2 Probability Mass & Density Functions UNDERSTAND
IPL APPLICATION · RUNS PER DELIVERY / WIN PROBABILITY

The PMF of runs per over gives P(X=k). From deliveries.csv we compute the empirical PMF for each team. The PDF of the Beta distribution gives the density of win probability θ.

// Poisson PMF (runs per over): P(X = k) = e^-λ × λ^k / k! // Beta PDF (win probability θ): f(θ) = θ^(α-1) × (1-θ)^(β-1) / B(α,β) // Key properties: // PMF: Σ P(X=k) = 1 (sums to 1) // PDF: ∫ f(x)dx = 1 (area under curve = 1)
2.3 Distribution Functions (CDF) UNDERSTAND
IPL APPLICATION · CREDIBLE INTERVALS ON DASHBOARD

The CDF F(x) = P(X ≤ x) gives cumulative probability. In the predictor, it answers: "What is P(CSK scores fewer than 160 runs)?" and generates the credible interval shown on the win probability bar.

// Poisson CDF (score below target T): P(X ≤ T) = Σₖ₌₀ᵀ [e^-λ × λ^k / k!] // In R: ppois(T, lambda) // 95% Credible Interval from Beta CDF inverse: CI_low = qbeta(0.025, α, β) CI_high = qbeta(0.975, α, β) // This is the interval displayed on the dashboard
2.4 Joint Probability Mass & Density Functions UNDERSTAND
IPL APPLICATION · JOINT RUNS & WICKETS

A joint distribution describes two or more RVs together. Runs scored (X) and wickets lost (Y) are not independent — losing wickets restricts run-scoring in death overs.

// Joint PMF (discrete): P(X=x, Y=y) = P(runs=x AND wickets=y in innings) // Marginalisation to get runs distribution: P(X=x) = Σᵧ P(X=x, Y=y) // Joint PDF (continuous: run-rate λ, venue factor v): f(λ, v) — estimated from joint innings+venue data // The 3-model ensemble approximates this joint PDF
2.5 Joint Distribution Functions UNDERSTAND
IPL APPLICATION · P(TEAM1 SCORE > TEAM2 SCORE)

The joint CDF F(x,y) = P(X ≤ x, Y ≤ y) extends single-variable CDFs to multiple dimensions. It models the probability that both teams' scores fall below certain thresholds — the foundation for score-based win probability.

// Joint CDF (discrete): F(x,y) = P(X ≤ x, Y ≤ y) = Σᵢ≤ₓ Σⱼ≤ᵧ P(X=i, Y=j) // Properties: // F(∞,∞) = 1 // F(x,∞) = F_X(x) [marginal CDF of X] // F(-∞,y) = 0 // Win via scores: // P(Team1 wins) = P(Score1 > Score2) // = Σₓ P(Score1=x) × P(Score2 < x)
2.6 Marginal Probability Distributions APPLY
IPL APPLICATION · ISOLATING SINGLE-TEAM STATS

Given a joint distribution of (Team1 score, Team2 score), the marginal distribution of Team1's score is obtained by summing out Team2's scores — letting us study each team independently.

// Marginal PMF (discrete): P(X=x) = Σᵧ P(X=x, Y=y) // Marginal PDF (continuous): f_X(x) = ∫ f(x,y) dy // Applied: venue-specific marginal run rate f(λ_team1 | venue) = ∫ f(λ_team1, v_factor) dv // Gives team1 run-rate distribution marginalised over venue

The venue factor is derived from the marginal win rate of each team at each ground — computed by summing over all opponent strengths.

2.7 Conditional Probability Distributions APPLY
IPL APPLICATION · WIN RATE GIVEN VENUE

Conditional distributions P(X | Y=y) describe one variable's distribution given a specific value of another. This directly powers the venue factor and the Bayesian posterior.

// Conditional PMF: P(X=x | Y=y) = P(X=x, Y=y) / P(Y=y) // Applied: P(CSK wins | venue = Chepauk) // = CSK wins at Chepauk / total matches at Chepauk // Bayesian update is a conditional distribution: P(θ | data) ∝ P(data | θ) × P(θ) // Posterior win prob = P(θ | match outcomes observed)
2.8 Mathematical Expectation & Covariance APPLY
IPL APPLICATION · EXPECTED SCORES & MODEL CORRELATION

E[X] is the long-run average shown on the Expected Score cards. Covariance measures how two sub-models move together, which determines whether the ensemble variance is lower than each individual model.

// Expected score (Poisson): E[X] = λ × 20 // runs over 20 overs // Variance (Poisson: E = Var): Var[X] = λ × 20 // Ensemble variance with covariance: Var(w₁P₁ + w₂P₂ + w₃P₃) = Σ wᵢ²Var(Pᵢ) + 2Σᵢ<ⱼ wᵢwⱼCov(Pᵢ,Pⱼ) // Negative covariance → ensemble more stable than any single model
2.9 Moment Generating Function APPLY
IPL APPLICATION · DERIVING POISSON PROPERTIES

The MGF M_X(t) = E[e^(tX)] uniquely characterises a distribution and generates all moments by differentiation. The Poisson MGF validates the key property E[X] = Var[X] = λ that we rely on.

// Poisson MGF: M_X(t) = exp(λ(eᵗ - 1)) // 1st moment (mean): E[X] = M'(0) = λ ✓ // 2nd moment → Variance: E[X²] = M''(0) = λ² + λ → Var[X] = λ ✓ // We verify E=Var empirically from deliveries.csv // before applying Poisson model

Correlation and Regression
CO: 3 · Relationship Modelling & Prediction
3.1 Introduction to Correlation UNDERSTAND
IPL APPLICATION · BATTING AVG VS WIN RATE

Correlation measures the linear association between two variables. Key questions: Does a team's batting average correlate with its win rate? Does powerplay run-rate predict final score?

// Pearson correlation coefficient: r = Σ[(xᵢ-x̄)(yᵢ-ȳ)] / √[Σ(xᵢ-x̄)² × Σ(yᵢ-ȳ)²] // r ∈ [-1, 1] // Interpretations for IPL data: // r(batting_avg, win_rate) ≈ +0.72 → strong positive // r(bowling_econ, win_rate) ≈ -0.68 → strong negative // Strong correlations justify including a feature in the model
3.2 Correlation and Rank Correlation APPLY
IPL APPLICATION · VALIDATING PLAYER IMPACT SCORE RANKINGS

Spearman's rank correlation is used when data is non-normal or ordinal. We use it to verify that our computed impact score ranks players consistently with batting average rankings.

// Spearman's ρ: ρ = 1 - [6 × Σdᵢ²] / [n(n²-1)] // dᵢ = difference in ranks of player i // ρ ≈ 0.85 → impact score ranks well-correlated with batting avg // Kendall's τ (alternative): τ = (Concordant - Discordant) / [n(n-1)/2]

High rank correlation between our impact score and expert cricket ratings validates that the composite formula is a good proxy for true player quality without needing labelled training data.

3.3 Regression APPLY
IPL APPLICATION · PREDICTING SCORE FROM POWERPLAY

Simple linear regression models final innings score as a linear function of powerplay run-rate (first 6 overs), providing a complementary score predictor to the Poisson model.

// Regression model: ŷ = β₀ + β₁x // y = final score, x = powerplay run-rate // OLS estimators (minimise sum of squared residuals): β₁ = Σ(xᵢ-x̄)(yᵢ-ȳ) / Σ(xᵢ-x̄)² β₀ = ȳ - β₁x̄ // R² (proportion of variance explained): R² = 1 - (SS_residual / SS_total) // R² ≈ 0.65 → powerplay explains 65% of final score variance
# R implementation model <- lm(final_score ~ powerplay_rr, data=innings_data) summary(model)
3.4 Partial & Multiple Correlation APPLY
IPL APPLICATION · ISOLATING VENUE EFFECT

Partial correlation measures the relationship between win probability and venue after controlling for team strength — isolating the true venue effect free of confounding.

// Partial correlation r(X,Y | Z): r_XY.Z = (r_XY - r_XZ × r_YZ) / √[(1-r_XZ²)(1-r_YZ²)] // r(win_prob, venue | team_strength) // Controls for opponent quality before measuring venue effect // Multiple R (3-model ensemble): R² = Var(ensemble prediction) / Var(actual outcomes)
3.5 Multiple Regression APPLY
IPL APPLICATION · DATA-DRIVEN ENSEMBLE WEIGHTS

Multiple regression extends the ensemble to learn optimal weights from data instead of the hardcoded 40/35/25 split. This would make our model statistically optimal rather than heuristic.

// Multiple regression model: P(win) = β₀ + β₁×P_bayesian + β₂×P_poisson + β₃×P_venue + ε // OLS matrix form: β̂ = (XᵀX)⁻¹ Xᵀy // Adjusted R² (penalises extra predictors): R̄² = 1 - [(1-R²)(n-1)/(n-k-1)]

A future version of the predictor would replace the hardcoded ensemble weights with OLS-estimated β coefficients, making the model fully data-driven and extensible to new predictors.

Probability Distributions
CO: 2 · Classical Distributions Applied to Cricket Data
4.1 Probability Distributions — An Introduction UNDERSTAND
IPL APPLICATION · CHOOSING THE RIGHT DISTRIBUTION

The predictor uses four different distributions, each chosen because its mathematical properties match the physical process being modelled. Distribution selection is the most critical modelling decision.

  • Binomial → match outcomes (n independent win/loss trials)
  • Poisson → runs per over (discrete events at constant rate)
  • Beta → win probability θ (continuous, bounded [0,1])
  • Normal → large-sample approximations via CLT
  • Exponential / Gamma → time between wickets falling
  • Weibull → non-constant hazard rate across overs
4.2 Binomial Distribution APPLY
IPL APPLICATION · H2H WIN RECORDS & MATCHUP TEST

The Binomial(n,p) distribution models the number of wins in n matches with fixed win probability p. It is the likelihood model in Bayesian updating and the basis of the matchup significance test.

// PMF: P(X=k) = C(n,k) × pᵏ × (1-p)^(n-k) // Mean: E[X] = np // Variance: Var[X] = np(1-p) // Matchup test (e.g. Bumrah vs Kohli): // H₀: p = 0.12 (overall wicket rate) P(X ≥ 3 | n=8, p=0.12) = 1 - pbinom(2, 8, 0.12) // p-value < 0.05 → Bumrah dominates this matchup ★
4.3 Poisson Distribution APPLY
IPL APPLICATION · CORE RUN-RATE MODEL

The Poisson(λ) distribution is the core of the Expected Score model. Runs in an over are discrete events arriving at a constant average rate — the classic Poisson setting.

// PMF: P(X=k) = e^-λ × λ^k / k! // Key: E[X] = Var[X] = λ // MLE of λ (Maximum Likelihood Estimate): λ̂ = x̄ = mean runs per over from innings data // Expected T20 score: E[Score] = λ̂ × 20 // Win probability from run-rates: P(A wins) ≈ λ_A / (λ_A + λ_B)
# MLE and expected score lambda <- mean(team_innings$runs_per_over) expected_score <- lambda * 20
4.4 Normal Distribution APPLY
IPL APPLICATION · CLT & LARGE-SAMPLE INFERENCE

By the Central Limit Theorem, the sample mean run-rate x̄ is approximately Normal for large n. This underpins the standard error calculations used to build model confidence intervals.

// Normal approx to Binomial (n large): X ~ Bin(n,p) ≈ Normal(μ=np, σ²=np(1-p)) // 95% CI for win probability p̂ (Wald interval): p̂ ± 1.96 × √[p̂(1-p̂)/n] // Bayesian CI using Normal approx of Beta posterior: CI ≈ p̂ ± 1.96 × SE SE = √[αβ / ((α+β)²(α+β+1))] // Displayed as the credible interval on the dashboard
4.5 Exponential Distribution APPLY
IPL APPLICATION · BALLS BETWEEN WICKETS

The Exponential(μ) distribution models waiting time between events in a Poisson process. In cricket: the number of balls between consecutive wickets follows an exponential distribution.

// PDF: f(x) = μ × e^(-μx), x ≥ 0 // Mean time between wickets: E[X] = 1/μ // P(next wicket within 6 balls): P(X ≤ 6) = 1 - e^(-μ×6) // Memoryless property: // P(wicket next ball | survived k balls) = P(wicket next ball) // Justifies treating each delivery as an independent trial // in the Binomial matchup test
4.6 Gamma Distribution APPLY
IPL APPLICATION · BALLS UNTIL k WICKETS FALL

The Gamma(k,μ) distribution models time until the k-th event in a Poisson process — i.e., balls until a team loses k wickets. It is also the conjugate prior for the Poisson rate λ.

// PDF: f(x) = [μᵏ x^(k-1) e^(-μx)] / Γ(k) // Mean: E[X] = k/μ // Variance: Var[X] = k/μ² // Applied: balls until 5 wickets fall // Gamma(k=5, μ=team's wicket rate per ball) // E[X] = 5/μ // Also: Gamma-Poisson model (alternative to Beta-Binomial) // λ ~ Gamma(α,β) → robust run-rate prior
4.7 Weibull Distribution APPLY
IPL APPLICATION · NON-CONSTANT HAZARD BY OVER

The Weibull(k,λ) generalises Exponential by allowing the hazard rate to change over time. Wicket-taking rates differ between powerplay, middle overs, and death overs — a non-constant hazard structure.

// PDF: f(x) = (k/λ)(x/λ)^(k-1) e^(-(x/λ)^k) // Shape parameter k determines hazard behaviour: // k < 1 → decreasing hazard (batsmen settle in, early overs) // k = 1 → constant hazard (Exponential, middle overs) // k > 1 → increasing hazard (tail-enders, death-over risks) // Hazard function: h(x) = (k/λ) × (x/λ)^(k-1) // Directly connects to Module 7 (Reliability)

The Weibull model bridges Modules 4 and 7: wicket-fall timing in cricket follows exactly the same mathematical framework as component failure in reliability engineering, with cricket overs as the "time" axis.

Hypothesis Testing – I
CO: 4 · Large Sample Tests · Z-Tests for Proportions and Means
5.1 Hypothesis Testing — An Introduction UNDERSTAND
IPL APPLICATION · FORMAL STATISTICAL CLAIMS

Every statistical statement in the predictor can be framed as a hypothesis test. The matchup table's ★ symbol literally represents a rejected null hypothesis at α = 0.05.

// Framework: H₀ (Null): "Pattern is due to chance" H₁ (Alternative): "Pattern is statistically real" // Decision rule: p-value = P(result this extreme | H₀ true) // Reject H₀ if p-value < α = 0.05 → flag ★ in matchup table // Error types: // Type I (α): False positive — flag a neutral matchup as dominant // Type II (β): False negative — miss a real dominance pattern // Power = 1 - β (probability of detecting a real effect)
5.2 Types of Errors, Critical Region & Large Sample Tests UNDERSTAND
IPL APPLICATION · MATCHUP DOMINANCE THRESHOLD

We must balance Type I (false positive matchup flags) and Type II (missed real dominance) errors. The critical region defines the boundary where we call a matchup "dominant."

// Critical region (one-sided binomial, exact): Reject H₀ if X ≥ c where P(X ≥ c | H₀) ≤ 0.05 // Large sample (Z-test) approximation when n > 30: Z = (X - n×p₀) / √(n×p₀×(1-p₀)) // One-sided: Reject if Z > 1.645 (α = 0.05) // Two-sided: Reject if |Z| > 1.96 (α = 0.05) // Predictor uses one-sided test (directional hypothesis) // → more power to detect real effects
5.3 Z-Test for Single & Difference of Proportions APPLY
IPL APPLICATION · IS A TEAM'S WIN RATE ABOVE CHANCE?

Z-tests for proportions formally test whether a team's observed win rate is significantly above 50%, and whether two teams have significantly different win rates at a specific venue.

// Single proportion (Is CSK win rate > 0.5?): Z = (p̂ - p₀) / √(p₀(1-p₀)/n) // p̂ = observed rate, p₀ = 0.5, n = matches played // Two-proportion test (CSK vs MI win rate at Wankhede): p̄ = (x₁+x₂) / (n₁+n₂) // pooled proportion Z = (p̂₁ - p̂₂) / √[p̄(1-p̄)(1/n₁ + 1/n₂)]
prop.test(x=csk_wins, n=total_matches, p=0.5, alternative="greater")
5.4 Z-Test for Single Mean & Difference of Means APPLY
IPL APPLICATION · ARE TWO TEAMS' RUN-RATES DIFFERENT?

We test whether two teams have significantly different mean run-rates — statistically justifying why the Poisson model assigns them different λ values rather than a shared parameter.

// Single mean (Is CSK λ > 8.0 runs/over?): Z = (x̄ - μ₀) / (σ/√n) // Difference of means (CSK λ vs MI λ): Z = (x̄₁ - x̄₂) / √(σ₁²/n₁ + σ₂²/n₂) // σ estimated by s when n is large (CLT applies) // Reject H₀: μ₁=μ₂ if |Z| > 1.96 (two-sided, α=0.05) // If rejected → different λ values are statistically justified

Hypothesis Testing – II
CO: 4 · Small Sample Tests · ANOVA · Design of Experiments
6.1 Hypothesis Testing II — An Introduction UNDERSTAND
IPL APPLICATION · SMALL SAMPLE SITUATIONS IN CRICKET

When n < 30, or when σ² is unknown, Z-tests are unreliable. Player matchup data often has very few observations — e.g., Bumrah has bowled only 8 overs to Kohli. Module 6 covers exact and small-sample methods appropriate for this regime.

This is precisely why the predictor uses the exact Binomial test (not the Z approximation) for player matchup significance — the sample sizes are too small for the normal approximation to be valid.

6.2 Small Sample Tests — Student's t-Test APPLY
IPL APPLICATION · NEW PLAYERS WITH LIMITED IPL DATA

When a new player has only 12 IPL innings, the t-test gives valid inference without knowing σ. We test whether the debutant's run-rate is significantly above the league average.

// One-sample t-test (n small, σ unknown): t = (x̄ - μ₀) / (s/√n) ~ t_{n-1} // t has heavier tails than Z → wider CIs → more conservative // As n→∞, t_{n-1} → Normal(0,1) // Two-sample t-test (compare two players' run-rates): t = (x̄₁ - x̄₂) / (s_p × √(1/n₁ + 1/n₂)) // s_p = pooled SD, df = n₁+n₂-2
t.test(player_runrates, mu=7.5, alternative="greater")
6.3 F-Test APPLY
IPL APPLICATION · DO VENUES HAVE EQUAL SCORE VARIABILITY?

The F-test compares variances of two populations. High variance differences between venues violate the equal-variance assumption needed for pooled t-tests and some ANOVA formulations.

// F-test statistic: F = s₁² / s₂² ~ F_{n₁-1, n₂-1} // H₀: σ₁² = σ₂² (Wankhede and Chepauk equally variable) // Reject if F > F_critical or F < 1/F_critical // F also appears in regression overall significance: // F = MS_regression / MS_residual // Tests: are any predictors (P_bayesian, P_poisson, P_venue) // significantly associated with match outcomes?
6.4 Chi-Square Test — Goodness of Fit APPLY
IPL APPLICATION · VALIDATING THE POISSON ASSUMPTION

Before applying the Poisson model, we must verify that runs-per-over actually follow a Poisson distribution. The χ² goodness-of-fit test is the formal check.

// Chi-Square statistic: χ² = Σ [(Oᵢ - Eᵢ)² / Eᵢ] ~ χ²_{k-1-p} // Oᵢ = observed count in bin i // Eᵢ = expected count under Poisson(λ̂) // p = 1 (one estimated parameter: λ) // H₀: data follows Poisson → Reject if χ² > χ²_critical
# Goodness-of-fit test in R obs <- table(runs_per_over) exp <- dpois(0:20, lambda=lambda_hat) * n chisq.test(obs, p=exp/sum(exp))
6.5 Chi-Square Test — Independence of Attributes APPLY
IPL APPLICATION · IS TOSS RESULT INDEPENDENT OF MATCH OUTCOME?

Is winning the toss associated with winning the match? The χ² independence test on a 2×2 table answers this — and the typical result (p > 0.05) justifies why toss is excluded from our model.

// 2×2 Contingency Table: // Win Match Lose Match // Win Toss: a b // Lose Toss: c d // Expected cell frequency: Eᵢⱼ = (row total × col total) / grand total // Test statistic: χ² = Σ [(Oᵢⱼ - Eᵢⱼ)² / Eᵢⱼ] ~ χ²_{(r-1)(c-1)} // p > 0.05 → toss and match result are independent // → toss not included in ensemble (justified!)
6.6 Design of Experiments — CRD, RBD and LSD APPLY
IPL APPLICATION · CONTROLLING CONFOUNDERS IN MATCH DATA

IPL match data is observational. DOE concepts help identify and control sources of variation — venues and seasons are confounders that bias naïve comparisons of team strength.

  • CRD: Compare team run-rates ignoring venues — valid only if venues are randomly assigned (not the case)
  • RBD: Block by venue — venues as blocks controlling for pitch conditions. Gives fairer team comparisons
  • LSD: Control for both venue and season simultaneously — each team plays each venue each season (approximately true in IPL)

Key insight: Naively comparing two teams' win rates ignores venue confounding. RBD-style analysis — blocking on venue — powers the venue-adjusted win probabilities in our model.

6.7–6.9 Analysis of Variance (One, Two & Three Way) ANALYZE
IPL APPLICATION · DO VENUES / SEASONS SIGNIFICANTLY AFFECT SCORES?

One-Way ANOVA: Tests if mean scores differ across the 10 IPL venues. Two-Way ANOVA: Simultaneously tests venue and season effects. Three-Way ANOVA: Adds team as a third factor.

// ANOVA decomposition: SS_total = SS_between + SS_within // F-ratio (One-Way, k groups): F = [SS_between/(k-1)] / [SS_within/(N-k)] // H₀: all 10 venue mean scores are equal // Large F → venues significantly differ → venue factor justified // Two-Way: // SS_total = SS_venue + SS_season + SS_interaction + SS_error
model <- aov(score ~ venue, data=matches_df) summary(model) # F-statistic, p-value TukeyHSD(model) # Which venue pairs differ?

If ANOVA rejects H₀ (p < 0.05), it statistically justifies the venue factor weight (25%) in the ensemble. The Tukey post-hoc test identifies which specific venues are outliers — informing the venue adjustment parameters.

Reliability
CO: 5 · System Reliability, Maintainability & Availability
7.1 Reliability — Introduction and Motivation REMEMBER
IPL APPLICATION · TEAM & PLAYER CONSISTENCY

Reliability R(t) is the probability that a system performs its intended function for a specified period under stated conditions. In cricket: a team's reliability is how consistently it performs to its benchmark — not just its average.

A team with R = 0.80 has an 80% probability of performing to its benchmark standard in any given match. High-reliability teams have low variance and high mean run-rates — our model has tighter confidence intervals for them.

Why reliability matters for prediction: Standard statistics (means, win rates) are time-independent. Reliability adds the temporal dimension — critical for understanding form, fatigue, and squad depth across a tournament.

7.2 Basic Concepts — Hazard Function UNDERSTAND
IPL APPLICATION · WICKET HAZARD RATE BY OVER

The hazard function h(t) gives the instantaneous rate of "failure" given survival to time t. In cricket: t = over number, failure = losing a wicket. The hazard rate is not constant — it follows a "bathtub" curve.

// Hazard function: h(t) = f(t) / R(t) // f(t) = PDF of time-to-failure (time to next wicket) // R(t) = 1 - F(t) = survival function // Cricket "bathtub" curve: // Overs 1–6 (Powerplay): h(t) decreasing — batsmen cautious // Overs 7–15 (Middle): h(t) constant — settled partnerships // Overs 16–20 (Death): h(t) increasing — aggressive, risky shots // Cumulative hazard: H(t) = -ln R(t) = ∫₀ᵗ h(u) du
7.3 Series & Parallel Systems — System Reliability APPLY
IPL APPLICATION · BATTING LINEUP & BOWLING ATTACK MODELS

A cricket batting lineup is a series system: all key contributors must perform for the total to be competitive. The bowling attack is a parallel system: at least one bowler must perform well to restrict the opposition.

// Series system (batting lineup, n key batsmen): R_lineup = R₁ × R₂ × … × Rₙ // Each batsman: Rᵢ = 0.70 → 6 batsmen: // R_lineup = 0.70⁶ ≈ 0.12 (chain breaks at weakest link) // Parallel system (bowling attack, m bowlers): R_attack = 1 - (1-R₁)(1-R₂)…(1-Rₘ) // Each bowler: Rᵢ = 0.60 → 4 bowlers: // R_attack = 1 - 0.40⁴ ≈ 0.97 (redundancy gives robustness)

This explains why balanced batting lineups (multiple reliable contributors) consistently outperform star-dependent teams — the series reliability product degrades rapidly without batting depth.

7.4 Maintainability — Preventive & Repair Maintenance ANALYZE
IPL APPLICATION · RECOVERY AFTER WICKETS FALL

Maintainability M(t) is the probability a failed system is restored within time t. In cricket: how quickly can a team recover after losing early wickets? The middle order acts as the "repair component."

// Maintainability (Exponential repair times): M(t) = 1 - e^(-μt) // μ = repair rate (scoring rate of middle order per over) // t = remaining overs available // Applied: P(team recovers from 50/3 to 150+ in 14 overs) // μ = middle-order run-rate after collapse // Preventive maintenance → player rotation policy: // Resting a key bowler (preventive) = avoid failure before it occurs // Bowling them to exhaustion (run to failure) = corrective maintenance // Optimal policy: maximise team availability over 14-match season
7.5 Availability ANALYZE
IPL APPLICATION · PLAYER AVAILABILITY & SQUAD MANAGEMENT

Availability A is the long-run proportion of time a system is operational. For players, it accounts for injuries, suspensions, and fatigue — the critical missing component from our current predictor.

// Steady-state availability: A = MTTF / (MTTF + MTTR) // MTTF = Mean Time To Failure (avg matches between injuries) // MTTR = Mean Time To Repair (avg matches to return) // Example: MTTF=10 matches, MTTR=3 matches A = 10/(10+3) ≈ 0.77 // Player available for 77% of matches ≈ 11 of 14 IPL games // Team availability (series model): A_team = Π Aᵢ for i = 1 to 11 // Expected available squad strength per match

The predictor's listed limitation "No player injury or availability information" is precisely a missing availability model. Adding MTTF/MTTR from historical injury records would enable dynamic team strength that degrades during long tournaments — a direct and powerful application of Module 7 to cricket analytics.
